## Ticket

## Goal

Adopt **[Keep a Changelog](https://keepachangelog.com/)** format and a **semver** tagging practice for releases.

## Deliverables

- [x] **`CHANGELOG.md`** at repo root.
- [x] Process note in README: when to add `## [Unreleased]` entries.

## Steps

1. Seed `CHANGELOG.md` with `## [Unreleased]` and first version `0.1.0` if appropriate.
2. Categories: Added, Changed, Fixed, Removed, Security.
3. Tag releases on GitHub matching semver (`v0.1.0`).

## Files / paths

- **Create:** `CHANGELOG.md`
- **Edit:** [README.md](../../README.md)

## Acceptance criteria

- At least one version section exists; future PRs update Unreleased.

## Out of scope
